※ 개발 단계에 따른 애플리케이션 테스트

1. 단위 테스트
	- 코드 작성 직후에 테스트
	- 요구사항을 기반으로 한 기능성 테스트를 최우선한다.
	ex) 명세 기반 테스트(블랙박스 테스트), 구조 기반 테스트(화이트박스 테스트)

2. 통합 테스트
	- 단위 테스트를 거친 모듈들을 한 시스템으로 결합하는 과정에서 테스트
	- 모듈 간 또는 통합된 컴포넌트들 간의 상호작용 오류를 확인
	ex) 빅뱅 통합 테스트, 하향식 통합 테스트(스텁), 상향식 통합 테스트(드라이버, 클러스터), 혼합식 통합 테스트

3. 시스템 테스트
	- 개발된 소프트웨어가 컴퓨터 시스템 상에서 완벽히 수행되는가 확인
	- 실제 사용환경과 유사한 환경을 만들어 테스트한다.
	ex) 기능적 요구사항, 비기능적 요구사항

4. 인수 테스트
	- 사용자가 직접하는 테스트
	- 요구사항을 충족하였는지 확인
	ex) 사용자 인수 테스트, 운영상의 인수 테스트, 계약 인수 테스트, 규정 인수 테스트, 알파 테스트, 베타 테스트
